PR #2232 uses a promise-chain map keyed by resolved `git-common-dir` for status and local index/worktree mutations. It also cannot provide a cross-process guarantee.

## Implemented invariants

- Canonical common/worktree identities cover subdirectories, linked worktrees, symlinks, relative common-dir output, Windows case identity, and confirmed non-repositories.
- Discovery is injected, canonical single-flight, separate from execution scheduling, capped at two active tasks, and bounded at every queue/map.
- Reads and writes conflict by worktree; distinct linked-worktree local work may progress; common writes serialize with each other and topology writes are barriers; earlier conflicting writers block later reads.
- Top-level execution, per-common reads, context/global queues, contexts, worktrees, status in-flight work, and retained idle state are explicitly bounded.
- Queue cancellation is explicit; running mutations settle normally. Compatible compound work can reuse a lease and incompatible re-entry fails instead of deadlocking.
- Mutation generations advance on admission and completion/failure/cancellation.
- Status coalescing is in-flight only, generation-aware, full-to-light directional, failure-cleaned, and waiter-cancellation-safe.
- Read-class Git clients and direct read helper subprocesses receive operation-scoped `GIT_OPTIONAL_LOCKS=0`; mutations do not receive the override.
- Existing route/API response envelopes and web/Electron ownership are unchanged; no internal controls are exposed.
- Every exported service operation is classified as bootstrap/pure/memory/global-read/read/worktree-write/common-write/common+target/topology/clone.
- Common writes serialize shared mutations without becoming total barriers; topology writes remain common-context barriers.
- Network capacity is capped at one per common context and two globally; clone reservations share the global cap and remain destination-exclusive after releasing network capacity.
- Queue timeout, admission-time idle pruning, lease coverage, and selective topology worktree invalidation are explicit and tested.
- FS clone and both FS ignore-check paths delegate to the service; skills catalog uses clone reservations; notification branch lookup uses light status.
- External lock failures propagate unchanged enough to retain Git's authoritative lock diagnosis.
