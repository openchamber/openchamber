## Final before/after evidence

### Representative target — both run orders

- Both implementations passed 1,066 correctness checks per run, zero unclassified launches, and cleanup.
- Git launches: 5,220 before versus 3,960 after — 1,260 / 24.138% fewer after.
- Before-first duration: 11,548.750ms before versus 16,540.896ms after — after 43.227% longer.
- After-first duration: 10,596.833ms before versus 15,551.758ms after — after 46.759% longer.
- Interpretation: one status per identity does not exercise coalescing. Historical broad burst parallelism completes sooner on this machine; current global/read/network bounds reduce launches and bound resource admission.

### Batched 30,000-caller fan-out — reverse order

- Both implementations passed 61,067 correctness checks, zero unclassified launches, and cleanup.
- Whole measured workload: 906,456.058ms before versus 27,888.368ms after — 32.503× faster after.
- Total Git launches: 215,220 before versus 5,760 after — 97.324% fewer after.
- Fan-out only: 895,880.638ms / 210,000 launches before versus 11,608.410ms / 1,800 launches after.
- Fan-out p95: 17,073.159ms before versus 236.470ms after — 72.200× lower after.
- Interpretation: repeated same-worktree/generation callers exercise the current in-flight sharing contract; the historical total FIFO repeats the service Git sequence per caller.
