# Рабочие процессы Git и GitHub

В OpenChamber есть встроенный git-представление, чтобы вы могли просматривать изменения, коммитить и управлять ветками без перехода в терминал. Откройте его на вкладке **Git** в правой боковой панели.

## Просмотр и коммит

Git-представление разделяет ваши изменения на **staged** и **unstaged**:

- нажмите **+** у файла, чтобы индексировать его, или **−**, чтобы убрать из индекса
- индексируйте или убирайте из индекса всю группу сразу
- нажмите на файл, чтобы увидеть его diff

Затем напишите сообщение коммита и закоммитьте. OpenChamber может **сгенерировать сообщение коммита** из ваших staged-изменений — он использует модель текущей сессии, поэтому нужно, чтобы сессия была открыта.

## Ветки и история

Git-представление покрывает и остальной повседневный git:

- создание, переключение, переименование и удаление веток
- push, pull и fetch
- просмотр истории и по-коммитных diff
- stash и восстановление изменений

## Pull-реквесты

Подключите GitHub (см. [Issues и PR GitHub](/github/)), и вкладка **PR** позволит открыть pull-реквест, обновить его, пометить готовым или слить — а также сгенерировать его заголовок и описание так же, как сообщения коммитов.

## Разрешение конфликтов

Если merge, rebase или integrate встречают конфликт, OpenChamber показывает, что застряло, и позволяет разрешить его — в том числе передать агенту.
