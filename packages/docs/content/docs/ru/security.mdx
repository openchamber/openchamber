# Безопасность

OpenChamber даёт доступ к вашей машине и коду, поэтому защитите его, прежде чем он станет доступен кому-то ещё. Эта страница описывает пароль UI, passkey и что нужно знать перед тем, как открывать OpenChamber в сеть.

## Задать пароль UI

Запустите OpenChamber с паролем, и браузерный UI запросит его:

```bash
openchamber --ui-password be-creative-here
```

Его также можно задать переменной окружения `OPENCHAMBER_UI_PASSWORD` вместо передачи в командной строке. После входа OpenChamber запоминает устройство на некоторое время, чтобы вас не спрашивали каждый раз.

Всегда задавайте пароль, если инстанс доступен кому-то ещё — особенно через [туннель](/tunnels/) или публичный интернет.

## Passkey

После задания пароля можно добавить passkey (Face ID, Touch ID, ключ безопасности) для быстрого входа. Добавляются в **Settings → OpenChamber → Passkeys**.

Passkey привязаны к текущему паролю. Если изменить или удалить пароль, сохранённые passkey сбрасываются, и их придётся добавить заново.

## Перед тем, как открывать доступ

- По умолчанию OpenChamber слушает только на вашей машине (`127.0.0.1`). Чтобы слушать шире, нужно явное изменение — и сначала задайте пароль.
- Предпочитайте [туннель](/tunnels/) или частную сеть (например, VPN) открытию порта в интернет.
- Если OpenChamber стоит за вашим HTTPS-сервером, см. [Обратный прокси](/reverse-proxy/).
