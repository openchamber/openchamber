# Удалённые инстансы

Desktop-приложение может подключиться к OpenChamber на другой машине через SSH — рабочему серверу, облачному узлу, домашней лаборатории — и вывести его UI на ваш экран, как если бы он был локальным. Настраивается в **Settings → Remote Instances**.

> Удалённые инстансы — функция **только для desktop**. В вебе или VS Code подключайтесь к удалённому серверу через переменные окружения из [OpenCode Server](/opencode-server/).

## Добавить удалённый инстанс

1. Откройте **Settings → Remote Instances** и добавьте инстанс.
2. Укажите SSH-команду, которой обычно пользуетесь для доступа к машине, и псевдоним.
3. Выберите, как OpenChamber там запускается:
   - **managed** — OpenChamber установит и запустит себя на удалённой машине
   - **external** — подключение к уже запущенному инстансу
4. Подключитесь.

OpenChamber проходит все этапы — проверка соединения, настройка удалённой машины, запуск сервера и проброс порта — и показывает текущий статус. При достижении статуса **ready** удалённый UI загружается локально.

## Учётные данные

Вы сами решаете, сохранять ли SSH- и UI-пароли или вводить их каждый раз. Если соединение разрывается, OpenChamber сообщает, на каком этапе произошёл сбой, чтобы вы могли его исправить — см. [Удалённый доступ](/troubleshooting/remote-access/).

## Прямые ссылки подключения

Если на удалённой машине уже запущен OpenChamber, создайте там ссылку подключения и импортируйте её в **Settings → Remote Instances → Server links**:

```bash
openchamber connect-url --port 3000 --server http://your-host:3000 --qr
```

`connect-url` сначала запускает сервер, если на этом порту ничего не работает. Добавьте `--api-only` для сервера без UI, `--lan` для привязки к LAN при запуске, `--ui-password` для защиты доступа через браузер и `--name` для подписи сохранённого подключения.

Сгенерированная ссылка содержит клиентский токен для приложений OpenChamber. Этот токен отделён от пароля браузерного UI и сохраняется между перезапусками сервера, пока вы его не отзовёте или не удалите.
