# Переменные окружения

OpenChamber считывает эти переменные окружения при запуске. Для стартовых сервисов `openchamber startup enable` по умолчанию делает снимок текущего окружения, поэтому перезапустите его после изменения переменных, которые должна использовать служба.

## Сервер OpenChamber

### `OPENCHAMBER_HOST`

Адрес привязки веб-сервера OpenChamber. Используйте `0.0.0.0`, чтобы разрешить доступ с других машин.

### `OPENCHAMBER_UI_PASSWORD`

Пароль для браузерного UI. Используйте при привязке вне localhost, использовании туннелей или запуске за обратным прокси.

### `OPENCHAMBER_API_ONLY`

Запускает OpenChamber в headless-режиме при значении `true` или `1`. API-маршруты остаются доступны для десктопных и мобильных клиентов, но браузерный UI не обслуживается.

### `OPENCHAMBER_DATA_DIR`

Переопределяет каталог данных OpenChamber. По умолчанию `~/.config/openchamber`.

### `OPENCHAMBER_COMPRESS_API`

Управляет сжатием ответов API. Используйте `true` или `1`, чтобы включить принудительно, `false` или `0`, чтобы выключить.

### `OPENCHAMBER_SKIP_API_COMPRESSION`

Отключает сжатие ответов API при значении `true` или `1`. Имеет приоритет над `OPENCHAMBER_COMPRESS_API`.

### `OPENCHAMBER_VERBOSE_REQUEST_LOGS`

Включает подробные логи HTTP-запросов при значении `true` или `1`.

### `OPENCHAMBER_UPDATE_API_URL`

Переопределяет endpoint API проверки обновлений. Большинству пользователей следует оставить это незаполненным.

### `OPENCHAMBER_PACKAGE_MANAGER`

Принудительно задаёт пакетный менеджер для операций обновления, когда автоопределение ошибочно.

## Сервер OpenCode

### `OPENCODE_HOST`

Подключает OpenChamber к существующему серверу OpenCode. Значение должно быть `http` или `https`-origin с явным портом и без пути, query или hash. `OPENCODE_HOST` имеет приоритет над `OPENCODE_PORT`.

### `OPENCODE_PORT`

Задаёт порт сервера OpenCode. С управляемым OpenCode запрашивает управляемый порт; с `OPENCODE_SKIP_START=true` подключается к внешнему серверу на этом порту.

### `OPENCODE_SKIP_START`

Предотвращает запуск собственного сервера OpenCode в OpenChamber при значении `true`.

### `OPENCHAMBER_OPENCODE_HOSTNAME`

Имя хоста привязки для сервера OpenCode, управляемого OpenChamber. По умолчанию `127.0.0.1`.

### `OPENCODE_BINARY`

Путь к исполняемому файлу `opencode`, который должен запускать OpenChamber.

### `OPENCODE_CONFIG`

Путь к конкретному файлу конфигурации OpenCode.

### `OPENCODE_CONFIG_DIR`

Путь к конкретному каталогу конфигурации OpenCode для агентов, навыков, сниппетов и обнаружения конфигурации.

### `OPENCODE_DATA_DIR`

Пользовательский каталог данных для управляемого сервера OpenCode.

### `OPENCODE_WSL_DISTRO`

Выбирает дистрибутив WSL, используемый для интеграции OpenCode на Windows.

### `OPENCHAMBER_OPENCODE_WSL_DISTRO`

Специфичный для OpenChamber алиас для выбора дистрибутива WSL. `OPENCODE_WSL_DISTRO` имеет приоритет, если заданы оба.

### `OPENCODE_JWT_SECRET`

Секрет, используемый для подписи токенов аутентификации UI. Используйте длинное случайное значение для постоянных развертываний службы.

## Терминал и Git

### `OPENCHAMBER_TERMINAL_SHELL`

Исполняемый файл оболочки для сессий терминала OpenChamber.

### `OPENCHAMBER_GIT_BINARY`

Исполняемый файл Git, используемый функциями Git в OpenChamber.

### `GIT_BINARY`

Альтернативное переопределение исполняемого файла Git. Для специфичной настройки OpenChamber предпочитайте `OPENCHAMBER_GIT_BINARY`.

### `OPENCHAMBER_GIT_READ_CACHE_TTL_MS`

Время жизни в миллисекундах для кэшированных чтений файлов на базе Git. Установите `0`, чтобы отключить этот кэш во время отладки.

## Голос и туннели

### `OPENAI_API_KEY`

API-ключ, используемый голосовыми функциями OpenChamber, которые вызывают OpenAI-совместимые сервисы.

### `OPENCHAMBER_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Разрешает удалённые OpenAI-совместимые базовые URL для голосовых функций при значении `true` или `1`.

### `NGROK_AUTHTOKEN`

Auth-токен ngrok, используемый командами туннелей OpenChamber. Также можно настроить ngrok через `ngrok config add-authtoken <token>`.

## Вспомогательные переменные окружения

### `BUN_BINARY`

Исполняемый файл Bun, который OpenChamber должен использовать при порождении процессов-демонов.

### `BUN_INSTALL`

Корень установки Bun. OpenChamber использует его для поиска `bin/bun` при запуске демона и обновлениях.

### `VITE_OPENCODE_URL`

Базовый URL API на этапе сборки для веб-приложения, собранного Vite. Большинству пользователей не следует задавать его для обычного CLI или десктопа.
