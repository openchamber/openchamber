# Multi-run

Multi-run запускает несколько сессий из одной формы — удобно для прогона одной задачи на разных моделях и сравнения результатов. Откройте его кнопкой вверху боковой панели сессий.

## Запуск multi-run

1. Откройте запуск multi-run.
2. Выберите проект и назовите группу запусков.
3. Напишите промпт и выберите модели (до пяти на группу).
4. Решите, нужно ли **изолировать запуски**.
5. Запустите.

Каждая модель получает собственную сессию, и все стартуют с вашим промптом.

## Изолированные запуски

Включите **isolate runs**, чтобы дать каждому запуску собственный [worktree](/worktrees/) и ветку, чтобы они никогда не касались одних и тех же файлов. Это требует git-репозитория — для каталогов без него опция автоматически выключается. Выберите ветку, от которой стартуют запуски.

Без изоляции каждый запуск — обычная сессия в каталоге проекта.

## Сравнение результатов

Каждый запуск — обычная сессия, которую можно открыть, прочитать, оставить или отбросить. Если вы запускали несколько для сравнения подходов, просмотрите их рядом и продолжите лучший.

Если один запуск не сможет стартовать, остальные всё равно запустятся — просто сессий будет меньше, чем вы запросили.
