# Worktrees и Git

Проблемы, которые возникают с [worktree-сессиями](/worktrees/) и [git-представлением](/git/), и как их решить.

## Worktree требует внимания

OpenChamber отмечает worktree, если что-то не так:

- **папка пропала** — папка worktree удалена или перемещена вне OpenChamber; удалите сессию и создайте новый worktree
- **ветка detached или unborn** — worktree не на нормальной ветке; переключите её на ветку
- **незавершённые merge, rebase или cherry-pick** — операция остановилась на полпути; завершите или отмените её из git-представления

## Не удаётся создать worktree

- **ветка уже существует** — выберите другое имя ветки или используйте вариант с существующей веткой
- **имя уже занято** — выберите другое имя worktree

## Не генерируется коммит или PR

Генерация сообщения коммита или описания PR выполняется в активной сессии, поэтому нужна открытая сессия с выбранной рабочей моделью. Откройте или выберите сессию и попробуйте снова.

## Проблемы с SSH или путями Windows

- убедитесь, что SSH-ключ репозитория — тот, что задан в вашей [git-identity](/git-identities/)
- на Windows git использует Unix-стиль путей (например, `/c/Users/...`) — OpenChamber это учитывает, но пользовательские пути SSH-ключей должны следовать тому же формату
