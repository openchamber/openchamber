# Подключение OpenCode

Если OpenChamber грузится, но не выходит за «OpenCode is restarting», или чат не отвечает, сервер недоступен. Пройдите по этим пунктам.

## Застряло на «OpenCode is restarting»

- дайте немного времени сразу после запуска — это состояние нормально, пока сервер стартует
- проверьте, жив ли сервер, командой `openchamber status`
- перезапустите через `openchamber restart`
- посмотрите детали запуска через `openchamber logs`

## Вы подключаетесь к своему серверу

Если OpenChamer настроен на существующий сервер, перепроверьте настройку из [OpenCode Server](/opencode-server/):

- `OPENCODE_HOST` должен включать порт без пути — например, `http://localhost:4096`
- установите `OPENCODE_SKIP_START=true`, чтобы OpenChamber не запускал свой собственный
- если адрес некорректен, OpenChamber игнорирует его и запускает свой сервер — ищите предупреждение `[config]` в логах

## По-прежнему не работает

- подтвердите, что Node.js версии `20` или новее
- переустановите последний CLI
- откройте `http://localhost:3000` напрямую, прежде чем тестировать туннель или удалённую ссылку
