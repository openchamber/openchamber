# Провайдеры, модели и агенты

Прежде чем OpenChamber сможет что-то сделать, нужно подключить хотя бы один AI-провайдер. Эта страница описывает подключение провайдера, выбор модели и настройку агентов.

## Подключить провайдера

1. Откройте **Settings → Providers**.
2. Откройте меню **Add provider** и выберите провайдера, который ещё не подключён.
3. Выполните вход одним из двух способов в зависимости от провайдера:
   - **API key** — вставьте ваш ключ и сохраните.
   - **Sign-in (device flow)** — OpenChamber покажет ссылку и короткий код. Откройте ссылку, введите код и подтвердите. OpenChamber завершит подключение сам.

Когда провайдер отображается как подключённый, его модели становятся доступны в чате.

Чтобы отключиться, откройте провайдера и выберите удаление его авторизации.

## Выбрать модель

Модель выбирается там, где вы работаете:

- в чате используйте выбор модели в строке сообщений, чтобы задать провайдера и модель для этой сессии
- для каждого агента можно задать модель по умолчанию (ниже)

## Настроить агентов

Агент — это именованная конфигурация: модель, характер и разрешённые действия.

1. Откройте **Settings → Agents**.
2. Выберите агента или создайте нового.
3. Отредактируйте любое из:
   - **description** — для чего нужен агент
   - **model** — его модель по умолчанию
   - **temperature** — насколько креативны его ответы
   - **prompt** — постоянные инструкции, которым он всегда следует
   - **tool rules** — какие инструменты он может использовать

## Где хранятся ваши авторизации

Авторизации провайдеров хранятся в OpenCode, а не в OpenChamber, поэтому они доступны и в OpenCode CLI. Если один и тот же провайдер задан в нескольких местах, выигрывает наиболее специфичная настройка: настройка проекта перекрывает личную.
