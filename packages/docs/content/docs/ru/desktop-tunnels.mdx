# Туннели в десктопе

Десктопное приложение может создать публичный туннель из **Настройки → OpenChamber → Туннель**. Для этого пути не нужно запускать OpenChamber из CLI.

## Установите провайдера

OpenChamber запускает CLI провайдера на вашей машине. Сначала установите провайдера, который хотите использовать:

```bash
brew install cloudflared
brew install ngrok
```

Cloudflare использует `cloudflared`. Ngrok требует аккаунт ngrok и authtoken из панели ngrok:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Запуск из приложения

1. Откройте **Настройки → OpenChamber → Туннель**.
2. Выберите **Cloudflare** или **Ngrok**.
3. Запустите быстрый туннель.
4. Отсканируйте сгенерированный QR-код с телефона.

Ngrok поддерживает только быстрые туннели. Cloudflare поддерживает быстрые туннели и управляемые режимы Cloudflare.

## Защита доступа

Даже когда URL провайдера сам по себе публичен, OpenChamber всё равно защищает доступ собственным connect-токеном. Сгенерированная connect-ссылка содержит одноразовый токен, имеет TTL, а старые неиспользованные ссылки отзываются при генерации новой или при остановке/перезапуске туннеля.
