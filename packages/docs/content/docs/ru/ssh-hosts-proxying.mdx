# SSH-хосты и проксирование

Используйте **Settings → Remote Instances** в desktop-приложении, чтобы импортировать сохранённые SSH-хосты, подключаться к удалённой машине и пробрасывать дополнительные порты через то же SSH-соединение.

> SSH-хосты и SSH-проксирование — функции **только для desktop**. Они используют SSH-клиент на вашем компьютере.

## Импортировать SSH-хост

OpenChamber умеет читать хосты из вашей локальной SSH-конфигурации — из того же места, что и команды вроде `ssh work-server`.

1. Откройте **Settings → Remote Instances**.
2. Выберите хост из **Saved SSH hosts**.
3. Если хост — шаблон, введите реальный адрес назначения, например `deploy@app.example.com`.
4. Сохраните и подключитесь.

OpenChamber создаст удалённый инстанс с этой SSH-командой. При достижении статуса **ready** удалённый UI OpenChamber откроется в desktop-приложении.

## Добавить дополнительные port forwards

У каждого удалённого инстанса есть раздел **Port Forwards**. Используйте его, когда что-то на одной стороне SSH-соединения должно достучаться до порта на другой стороне.

OpenChamber поддерживает три типа проброса:

- **Local (-L)** — открывает порт на вашем компьютере, который подключается к чему-то на удалённой машине.
- **Remote (-R)** — открывает порт на удалённой машине, который подключается обратно к вашему компьютеру.
- **Dynamic (-D)** — открывает локальный SOCKS-прокси через SSH-соединение.

Для большинства превью приложений и дашбордов на удалённой машине используйте **Local (-L)**.

## Использовать SOCKS-прокси

Выберите **Dynamic (-D)**, хотите, чтобы другие инструменты на вашем компьютере ходили в сеть через удалённую машину. OpenChamber открывает локальный SOCKS-прокси-порт для этого SSH-соединения.

После готовности соединения скопируйте или используйте локальный адрес прокси из строки проброса. Настройте браузер или инструмент на использование его как SOCKS5-прокси.

## Держите приватным

Используйте `127.0.0.1` или `localhost` для локальных адресов привязки, если только вы намеренно не хотите, чтобы другие устройства в сети имели доступ к проброшенному порту.
