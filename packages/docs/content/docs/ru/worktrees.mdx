# Worktree-сессии

Worktree-сессия выполняется в собственной выгрузке вашего репозитория, на собственной ветке (git worktree). Это не даёт параллельным сессиям мешать друг другу с файлами — одна может рефакторить, пока другая правит баг, без постоянного переключения.

## Создать worktree

1. Откройте диалог создания worktree кнопкой в верхней части боковой панели сессий.
2. Выберите стартовую точку:
   - **new branch** — задайте имя ветки и выберите, от какой ветки стартовать
   - **existing branch** — выгрузите уже существующую ветку
3. Подтвердите папку worktree (OpenChamber предлагает её по имени ветки).
4. Создайте.

OpenChamber создаёт ветку, настраивает папку и открывает в ней сессию. Запустить worktree можно также прямо из [задачи](/notes-todos-plans/) или из [GitHub issue или PR](/github/).

## Вернуть работу обратно

Когда работа готова, используйте **Integrate** в Git-представлении, чтобы перенести коммиты worktree на другую ветку (например, `main`). При конфликте изменений можно передать конфликт агенту для разрешения.

## Очистка

Удаление или архивация сессии может удалить worktree. Вы выбираете, удалять ли также ветку — локальную и, если есть, удалённую. Ничто не удаляется без вашего запроса.

## Если что-то не так

Worktree может потребовать внимания, если его папка пропала, ветка в состоянии detached или незавершённые merge/rebase. OpenChamber отмечает такие случаи, чтобы вы их исправили — см. [Worktree и Git](/troubleshooting/worktrees-git/).
