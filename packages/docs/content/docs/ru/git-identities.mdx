# Git-идентичности

Git-идентичность — это имя и email, которыми подписываются ваши коммиты. Если вы работаете в личных и рабочих репозиториях, можно сохранять идентичности и применять нужную к каждому репозиторию, вместо того чтобы полагаться на единую глобальную настройку. Управляйте ими в **Настройки → Git**.

## Добавьте идентичность

1. Откройте **Настройки → Git** и выберите **New**.
2. Введите **name** и **email** для коммитов.
3. Выберите способ аутентификации с удалённым репозиторием:
   - **SSH** — укажите на SSH-ключ
   - **token** — используйте сохранённые учётные данные для хоста
4. По желанию задайте цвет и значок, чтобы её было легко найти.

Глобальная идентичность вашей системы тоже показывается, только для чтения.

## Применение идентичности к репозиторию

Применение идентичности записывает её в **локальный** git-конфиг этого репозитория — она затрагивает только этот репозиторий, а не глобальную настройку. SSH-идентичности также задают SSH-команду для использования вашего ключа; token-идентичности настраивают хранилище учётных данных для хоста.

Вы можете импортировать идентичности, которые OpenChamber обнаруживает в существующих git-учётных данных, и сохранить их как token-идентичности.
