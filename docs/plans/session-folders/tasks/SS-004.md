### Handler `handleCreateNewFolder`

```
handleCreateNewFolder = useCallback(() => {
  if (!activeProjectId) return;
  const newFolder = createFolder(activeProjectId, 'New folder');
  // Ngay lập tức set rename mode cho folder mới
  setRenamingFolderId(newFolder.id);
  setRenameFolderDraft('New folder');
}, [activeProjectId, createFolder]);
```

> `setRenamingFolderId` và `setRenameFolderDraft` đã được thêm ở SS-001.

### Truyền rename state xuống SessionFolderItem

`SessionFolderItem` nhận props `isRenaming` và `renameDraft` để hiển thị inline rename ngay khi tạo:
- Khi `renamingFolderId === folder.id`: truyền `isRenaming={true}` và `renameDraft={renameFolderDraft}`
- Khi user save/cancel rename: `setRenamingFolderId(null)`

> Lưu ý: SS-003 render `SessionFolderItem` — executor của SS-004 cần thêm `isRenaming` và `renameDraft` props. Đây là coupling giữa SS-003 và SS-004. Executor làm SS-004 cần update lại props trong code của SS-003 nếu cần.

## Phần 2: Cleanup Effect

### Mục đích

Khi sessions bị xóa (qua `deleteSession` / `deleteSessions`), sessionIds trong folders có thể trở thành stale (không còn session nào). Cần cleanup định kỳ.

### Implementation

Thêm `useEffect` trong `SessionSidebar`:

```
useEffect:
  - Trigger khi: sessions array thay đổi (sessions là dependency)
  - Nếu activeProjectId rỗng: return
  - Lấy existingSessionIds = new Set(sessions.map(s => s.id))
  - Gọi: cleanupSessions(activeProjectId, existingSessionIds)
  
  Dependencies: [sessions, activeProjectId, cleanupSessions]
```

`cleanupSessions` trong store chỉ mutate nếu có thay đổi thực sự → không gây infinite loop.
